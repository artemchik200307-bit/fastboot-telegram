# FASTBOOT Telegram v1

Первый рабочий этап Telegram-интеграции.

## Уже работает

- пользовательский и административный боты в одном Render-сервисе;
- webhook для обоих ботов;
- безопасная привязка FASTBOOT ID + одноразовый код;
- постоянное пользовательское меню;
- балансы: Spot, AI Bot, Terminal, общий баланс;
- AI-статус и последние AI-сделки;
- пополнение через сумму + TXID;
- вывод с проверкой 50 USDT и периода 14 дней;
- реферальная статистика;
- административное меню;
- отправка событий в темы закрытой группы;
- уведомления о новых регистрациях;
- уведомления о пополнениях и выводах;
- персональные уведомления о закрытых AI-сделках;
- уведомления о сделках в административную тему.

## Важно

Никогда не добавляйте в GitHub:

- `SUPABASE_SERVICE_ROLE_KEY`;
- токены Telegram-ботов;
- реальные webhook secrets.

## Установка — по шагам

### 1. Supabase

Откройте:

`Supabase → SQL Editor → New query`

Вставьте целиком файл:

`supabase-telegram-v1.sql`

Нажмите `Run`.

### 2. GitHub

Создайте новый приватный репозиторий, например:

`fastboot-telegram`

Загрузите в него содержимое этого архива.

`.env.example` загружать можно: настоящих секретов в нём нет.

### 3. Render

Создайте:

`New → Blueprint`

Подключите репозиторий. Render прочитает `render.yaml`.

Добавьте Environment Variables из `.env.example`.

Обязательно укажите:

- `SUPABASE_URL`;
- `SUPABASE_SERVICE_ROLE_KEY`;
- `TELEGRAM_USER_BOT_TOKEN`;
- `TELEGRAM_ADMIN_BOT_TOKEN`;
- `WEBHOOK_BASE_URL`;
- два случайных webhook secret;
- `DEPOSIT_ADDRESS`.

### 4. WEBHOOK_BASE_URL

После первого создания Render покажет URL, например:

`https://fastboot-telegram.onrender.com`

Запишите его в `WEBHOOK_BASE_URL` и выполните Manual Deploy.

Сервис сам установит webhook при запуске.

### 5. Проверка

Откройте:

`https://ВАШ-СЕРВИС.onrender.com/health`

Должно быть:

`{"status":"ok"}`

Затем нажмите Start в обоих ботах.

## Что ещё нужно сделать на сайте

Пользовательскому боту нужен одноразовый код.

SQL-функция уже создана:

`telegram_generate_link_code()`

На следующем этапе мы добавим в Dashboard раздел Telegram и кнопку
«Получить код привязки».

Пока для теста код можно получить только через вызов RPC от
авторизованного сайта. Не создавайте коды вручную для чужих аккаунтов.

## Важное про getUpdates

После запуска webhook больше не открывайте `getUpdates` в браузере:
это уже не нужно. Webhook и ручной polling не должны использоваться
одновременно.
